E-Botar API Guide

Reference for all REST API endpoints when only the backend is accessible. Base URL: this server + /api.

Base URL

  • This server: /api (prepend your backend origin, e.g. https://your-api.com)
  • All endpoints below are relative to /api.

Authentication

Most endpoints require a JWT. Send it in the request header:

Authorization: Bearer <access_token>

Obtain token

POST /api/auth/token/
Content-Type: application/json

{"username": "your_username", "password": "your_password"}

Response: {"access": "...", "refresh": "..."}

Refresh token

POST /api/auth/token/refresh/
Content-Type: application/json

{"refresh": "<refresh_token>"}

Example requests

Register

POST /api/auth/register/
Content-Type: application/json

{"username":"student123","email":"student@snsu.edu.ph","password":"SecurePass123!","password_confirm":"SecurePass123!","first_name":"John","last_name":"Doe"}

Submit ballot

POST /api/voting/ballots/submit/
Authorization: Bearer <token>
Content-Type: application/json

{"election_id":1,"votes":[{"position_id":1,"candidate_id":5},{"position_id":2,"candidate_id":8}]}

Get election results

GET /api/voting/results/election_results/?election_id=1

Error responses

Errors are returned as JSON with a detail field (or field-specific errors):

{"detail": "Error message here"}
